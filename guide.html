Setup Guide

You don’t need coding experience for this workshop. We’ll do everything on github.com in your browser. Use the table of contents below whenever you want to jump ahead.

Built by Logan Jones with help from Cursor.

Before you start

Do these in order. Start Step 1 as early as possible. GitHub Education benefits can take about three days to activate after you’re accepted.

1 Create a GitHub account & apply for student benefits

If you don’t already have one, make a free account at github.com. Then apply for GitHub Education student resources:

Apply to GitHub Education as a student →

That free student pass gives you GitHub Pages hosting and other perks you’ll use for this project. Request it sooner rather than later: approval can take a few days, and benefits may not work immediately after acceptance.

AI coding help: This site was originally built with Cursor. Cursor Pro is no longer free for students, so we recommend GitHub Copilot (often included with GitHub Education) for help editing your site. See Using GitHub Copilot below. You can also paste the blue fill checklist into another AI coding assistant; see What to gather per layout.

2 Gather your content on your computer

Create a new folder on your desktop and collect what you want on your site so it’s easy to upload later:

  • Photos (headshot, project images, research figures)
  • Projects or research summaries (notes are fine; you’ll paste them into the template)
  • Your résumé or CV as a PDF
  • Anything else you want linked or featured

Pick a color palette before you edit CSS. Browse palettes at coolors.co (free, no account). Need a reference? See Logan’s plain document-style site: loganjones.org.

3 Optional: install a code editor

You can edit everything in your browser on GitHub (no install required). If you want a desktop editor (helpful with Copilot or local previews), choose one:

  • Visual Studio Code (lightweight, widely used): Windows · Mac
  • Cursor: a VS Code–based editor with AI help (what Logan used originally; student Pro is no longer free, but the app still works for basic editing)

Skip this step if you’re only using github.com in your browser. That’s enough to finish the workshop.

3 Turn on GitHub Pages (free hosting)

  1. In your new repo, click Settings (top tab).
  2. Click Pages in the left sidebar.
  3. Under Build and deployment → Source: pick Deploy from a branch.
  4. Branch: main · Folder: / (root) → click Save.
  5. Wait 1–2 minutes. Your URL appears at the top of the Pages settings page:
    https://YOUR-USERNAME.github.io/YOUR-REPO-NAME/
⚠️ Important Do NOT preview your site by clicking index.html inside the repo. That shows unstyled HTML. Always use your GitHub Pages URL from Settings → Pages.

4 Edit your content on GitHub

  1. Go to your repo on github.com.
  2. Click index.html (your homepage; there is no themes/ folder).
  3. Click the pencil icon (✏️ Edit this file) in the top-right.
  4. Press Cmd+F (Mac) or Ctrl+F (Windows) and search: EDIT HERE
  5. Change each labeled spot to your real information.
  6. Scroll to the bottom → click Commit changes.

5 Add your photo

  1. In your repo, click the assets folder.
  2. Click Add file → Upload files.
  3. Drag in your photo (name it something simple like photo.jpg).
  4. Click Commit changes.
  5. Go back to index.html and search for ADD YOUR PHOTO HERE.
  6. Follow the instructions in that comment to uncomment the <img> line and update the filename.

Need a free photo? See the Free stock photos dropdown below.

6 Change your colors

  1. Use a tool from the Color palettes section below to pick colors you like.
  2. Copy the hex codes (they look like #2563eb).
  3. In your repo, open css/[your-layout].css (see table in Step 3).
  4. At the top, find CHANGE YOUR COLORS HERE.
  5. Paste your hex codes over the existing ones. Only change the color values.
  6. Check contrast with a tool from Contrast checking before you commit.
  7. Click Commit changes.

7 Clean up when done

  • Delete the yellow trainer tip boxes from index.html (search for trainer-tip).
  • Delete the blue fill checklist when you've gathered everything (search for fill-checklist).
  • Update your links: search for EDIT LINK HERE and replace # with real URLs.
  • Hard-refresh your live site: Cmd+Shift+R or Ctrl+Shift+R.

Used a layout template repo? You're already set; no extra cleanup needed. See Slim down to one layout only if you copied the full demo repo by mistake.

🎨 Color palettes & generators

Generate a color scheme, then copy the hex codes into your css/[layout].css file.

Coolors.co

Free No account

The fastest palette generator. Press spacebar to shuffle colors. Click a color to copy its hex code.

  1. Go to coolors.co. A 5-color palette appears instantly.
  2. Press spacebar until you like the combination.
  3. Click any color → hex code copies to clipboard.
  4. Paste into --accent (or similar) in your CSS file.

Adobe Color Wheel

Free No account

Professional color harmonies: complementary, analogous, triadic, and more. Great when you have one color and need matching ones.

  1. Drag the circles on the wheel to pick a base color.
  2. Use the Color Harmony dropdown (left side) to try different schemes.
  3. Click any swatch below the wheel to copy its hex code.

HTML Color Codes: Picker

Free No account

Simple visual color picker. Click anywhere on the gradient, copy the hex. Good when you know the general color you want.

  1. Click on the color gradient to pick a shade.
  2. Copy the # hex code shown on the right.

Realtime Colors

Free No account

Preview how a full palette looks on a real website layout before you commit. Adjust text, background, and accent together.

  1. Edit the colors on the left panel.
  2. See a live preview on the right.
  3. Copy the hex codes when you're happy with the look.
♿ Contrast & accessibility checking

Text must be readable for everyone. Check that your text color has enough contrast against your background before publishing.

WebAIM Contrast Checker

Free No account

The gold standard. Enter your foreground (text) and background hex codes. Aim for AA or AAA pass.

  1. Paste your text color in Foreground Color.
  2. Paste your background color in Background Color.
  3. Check that "WCAG AA" shows Pass for normal text.
  4. If it fails, darken the text or lighten the background and try again.

Adobe Color: Contrast Analyzer

Free No account

Visual contrast checker from Adobe. Pick two colors on the wheel and see if they pass accessibility standards.

  1. Select two colors using the color pickers.
  2. Read the contrast ratio and pass/fail result below.

Colour Contrast Checker

Free No account

Minimal, fast checker. Type or paste two hex codes side by side and get an instant pass/fail.

🔌 How to plug assets into your site

Quick reference for using everything above in your portfolio template.

Photos & downloaded SVGs
  1. Download the file to your computer.
  2. On GitHub, go to your assets/ folder → Add file → Upload files.
  3. Drag the file in → Commit changes.
  4. In your theme HTML, reference it: <img src="../assets/yourfile.jpg" alt="Description">
Icons (copied SVG code)
  1. Copy SVG from Heroicons, Lucide, etc.
  2. In your theme HTML, paste the SVG code where you want the icon to appear.
  3. Add aria-hidden="true" to decorative icons, or aria-label="Email" if it's meaningful.
  4. To resize, add width="24" height="24" to the SVG tag.
Colors (hex codes)
  1. Generate or pick colors using Coolors or Adobe Color.
  2. Check contrast on WebAIM.
  3. Open css/[your-layout].css on GitHub.
  4. Replace hex values in the CHANGE YOUR COLORS HERE section.
  5. Commit → hard-refresh your live site.
Fonts (Google Fonts)
  1. Get the <link> embed code from fonts.google.com.
  2. Paste it in your theme HTML's <head> (near the other font links).
  3. In your CSS file, update font-family to match.

🔒 Privacy & safety: read this first

Your GitHub Pages site is public. Search engines can index it. Anyone with the URL can view it. Treat it like a billboard, not a private folder.

Never post:

  • Passwords, API keys, or login credentials
  • Social Security numbers, bank details, or government IDs
  • Your home address or personal phone number (use a university email instead)
  • Unpublished research data with identifiable participants (IRB restrictions apply)
  • Manuscripts under blind review that could identify you or co-authors
  • Lab-confidential or proprietary information
  • Anything you wouldn't hand to a stranger at a conference

Generally safe to post:

  • Published papers, preprints, and public conference posters
  • Professional headshots and public project screenshots
  • University email, LinkedIn, GitHub, and ORCID
  • Aggregated or anonymized results
  • Your CV (but review it first; CVs sometimes include more detail than you want public)

Every theme includes a red privacy notice box. Delete it only after you've reviewed your content, not before.

Images, video, PDF & audio

Each layout supports different media. Search your theme file for ADD to find every slot. The patterns below are built to stay fast (lazy-loaded images, no autoplay video) and accessible (alt text, captions, iframe titles).

🖼 Images (photos, screenshots, posters)
How to add any image
  1. Upload the file to your assets/ folder on GitHub (e.g. photo.jpg).
  2. Find ADD YOUR PHOTO HERE or ADD PROJECT IMAGE HERE in your theme HTML.
  3. Delete the placeholder <div> and uncomment the <img> line in the comment above it.
  4. Always write descriptive alt text (e.g. alt="Research poster on cognitive bias at APS 2025", not alt="image").
  5. Add loading="lazy" for images below the fold (already in template examples).

Tip: Compress large photos at squoosh.app (free, no account) before uploading. That keeps your site fast.

▶ Video (YouTube, Vimeo)
How to embed a video
  1. Upload your talk to YouTube or Vimeo (unlisted is OK; still don't share sensitive content).
  2. Click Share → Embed and copy the <iframe> code.
  3. In your theme HTML, find ADD VIDEO HERE.
  4. Paste the iframe inside a <div class="media-embed"> (see HTML comment in template).
  5. Set a descriptive title="..." on the iframe (required for screen readers).
  6. Delete the placeholder div. Never use autoplay.
📄 PDFs (CV, papers, slides)
How to link a PDF
  1. Upload cv.pdf (or similar) to assets/.
  2. Find ADD PDF HERE or a file-download link in your HTML.
  3. Update the href to ../assets/cv.pdf.
  4. Only upload PDFs you're allowed to share publicly.
🎧 Audio (podcast clips, interviews)
How to add audio (Minimal & Creative layouts)
  1. Upload .mp3 to assets/.
  2. Uncomment the <audio controls preload="none"> block in your HTML.
  3. Set src="../assets/yourfile.mp3".
  4. Native browser player: no account, accessible, and lightweight. No autoplay.

Slim down to one layout

Most students should skip this section. If you followed Step 2 and downloaded a solo ZIP, your repo already has one layout. Nothing to remove.

Recommended workflow: Preview on the live demo, then download the ZIP for your chosen layout. Don't copy the full template repo unless you're running the workshop demo.

Step-by-step on GitHub

  1. Pick your layout (e.g. Document: file is themes/document.html).
  2. Open index.html → pencil icon → select all → delete.
  3. Open themes/document.html → copy everything → paste into index.html.
  4. Fix paths in your new index.html (Find & Replace):
    • ../css/ → ./css/
    • ../js/ → ./js/
    • ../assets/ → ./assets/
    • ../guide.html → ./guide.html (or delete those links)
    • ../index.html → delete links to "All layouts"
  5. Remove the black layout switcher: delete the entire <nav class="theme-switcher">…</nav> block.
  6. Remove has-theme-switcher from the <body> class list.
  7. Delete training boxes: search trainer-tip and privacy-notice (keep privacy notice until you've reviewed content, then delete).
  8. Commit → your live URL now shows your portfolio at the root.

Performance & accessibility

These templates are meant to be fast and readable for everyone, including screen reader users and people who prefer reduced motion. Here's what that means in practice.

Performance: already built in

PracticeWhat we do
No frameworksPlain HTML/CSS (no React, no npm, no build step)
Small payloadsEach page loads ~2 CSS files + one 2 KB JS file (not all 10 themes at once)
Lazy imagesBelow-the-fold images use loading="lazy"
No autoplay videoVideos are embed-on-click or placeholders until you add them
System fonts on 5 themesDocument, Minimal, Terminal, Brutalist, and Deck skip Google Fonts, so there’s less to download on mobile
Compress your mediaYou: keep photos under ~500 KB; use JPG for photos, PNG for graphics

Accessibility: already built in

FeatureWhere
Skip to content linkPress Tab on any theme page; it jumps past the nav
Keyboard navigationAll links and buttons work without a mouse
Visible focus ringsTab through links and you'll see a clear outline
Alt text placeholdersEvery image example includes alt="…"; rewrite those for your photos
Video iframe titlesYouTube embeds require a title describing the video
Reduced motionAnimations (terminal cursor, scroll effects) turn off if the user prefers reduced motion
Color contrastText colors meet WCAG AA; still check after you change colors (see contrast tools)
Semantic HTML<main>, <nav>, <section>, heading order

Your checklist before publishing

  1. Every real image has descriptive alt text (not "image" or "photo").
  2. Video embeds have a title attribute on the <iframe>.
  3. After changing colors, run text/background through a contrast checker.
  4. Don't post private info. See the privacy section.
  5. Tab through your live site: can you reach every link and read every focus state?
  6. Delete yellow trainer-tip boxes when done.
